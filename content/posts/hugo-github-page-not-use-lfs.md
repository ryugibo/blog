# 변명

얼마전에 글을 쓰다가 이미지와 영상들을 넣을 일이 있어서 lfs를 활성화 해본 적이 있다.

활성화 했다가 다시 비활성화 한 이유는 다음과 같다.

## 이미지 영상 파일들을 버전 관리할 필요 없다.

기본적으로 작성한 글과 글에 첨부된 파일을 갱신하는 경우가 없다.

게임 프로젝트의 외부 에셋같은 경우에는 한 에셋을 계속 개선하면서 히스토리 파악이 필요하지만 글에 포함되는 첨부파일의 경우 갱신하는 경우가 있을까..?

있더라도 많지 않아서 해당 글에 대해서만 첨부파일 관련 히스토리를 날려주면 될것이라 판단됐다.

## macOS 기본 git에 lfs가 빠져있다.

이건 macOS에 git lfs를 설치하면 되는 부분인데, 개인적으로 기본 제공되는 프로그램이 있으면 되도록 손대지 않는 편이라서 이유에 넣었다.

macOS에서 ruby랑 git은 언제쯤 빠질까..🤔

# 결론

결론은 지극히 개인적인 이유.

github에서 개별 파일로 허용하지 않는 100MB를 넘는 이미지가 들어갈 일은 절대 없을테니 괜히 clone 속도만 저하시키는 lfs는 안쓰기로 했다.
